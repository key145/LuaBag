# 基于 XLua 的背包系统 Demo

## 📖 项目简介
一个使用 **XLua 热更新框架** 和 **AssetBundle 资源管理** 实现的背包系统演示。项目将所有核心逻辑（UI 交互、数据管理）用 Lua 实现，C# 仅负责资源加载和热更新环境初始化，真实模拟了商业游戏中的逻辑热更与资源动态加载流程。

## 🛠️ 技术栈
- Unity 2021 LTS
- XLua（热更新框架）
- Lua（逻辑层）
- AssetBundle（资源打包与加载）
- C#（底层框架）
- 单例模式、协程、委托

## ✨ 核心功能
- **纯 Lua 实现 UI 逻辑**：背包面板（`BagPanel.lua`）、物品格子（`ItemGrid.lua`）完全由 Lua 编写，通过 Toggle 切换装备、道具、宝石分类。
- **动态资源加载**：所有 UI 预制体、图集（SpriteAtlas）、Lua 脚本均通过 **AssetBundle** 动态加载，支持资源热更新。
- **模拟玩家数据**：`PlayerData.lua` 中模拟了装备、道具、宝石的初始数据，格子上显示对应图标和数量。
- **物品配置解析**：通过 Lua 版本的 JSON 库（`JsonUtility.lua`）解析物品 ID 与图标映射，实现图标动态绑定。
- **完整的 UI 管理**：主面板（`MainPanel.lua`）作为入口，点击按钮打开背包，Lua 层自行管理面板显隐。
- **C# 底层支持**：
  - `ABMgr`：负责 AB 包依赖加载、缓存和同步/异步资源加载。
  - `LuaMgr`：封装 XLua 环境，提供自定义 Loader 从 AB 包或文件系统加载 Lua 脚本。
  - `Main.cs`：启动入口，初始化 Lua 虚拟机并执行 `Main.lua`。

## 💡 实现要点
- **热更新架构**：逻辑层全部用 Lua 编写，更新时只需替换 Lua 脚本或 AB 包，无需重新打包客户端。
- **AB 包管理器**：自动加载依赖包，支持同步/异步加载，缓存已加载的 AB 包避免重复加载。
- **Lua 面向对象**：通过 `Object.lua` 基类实现 Lua 类的继承（如 `ItemGrid` 继承自 `Object`），代码结构清晰。
- **图集加载**：通过 `SpriteAtlas` 和 `ABMgr` 动态加载图集，根据物品 ID 解析图集名称和 Sprite 名称。
- **XLua 交互**：在 C# 中标记 `[CSharpCallLua]` 委托类型，支持 Lua 与 C# 事件相互调用。
- **资源分离**：UI 预制体、图集、Lua 脚本、JSON 配置均打成独立 AB 包，模拟真实项目资源管理。

## 🚀 如何运行
1. **克隆仓库**到本地。
2. **打开场景**。
3. **运行游戏**，点击主界面背包按钮，即可打开背包并切换分类查看物品。
